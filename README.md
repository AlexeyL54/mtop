mtop — это веб-ориентированный системный монитор для Linux, который в реальном времени отображает информацию о памяти, загрузке CPU и запущенных процессах через удобный веб-интерфейс.

## Возможности

- **Мониторинг памяти**: отображение общей, доступной, свободной и кэшированной памяти с графическим индикатором использования.
- **Мониторинг CPU**: загрузка каждого ядра процессора.
- **Список процессов**: детальная таблица с PID, именем, состоянием, виртуальной и физической памятью, а также количеством потоков.
- **Поиск и фильтрация**: мгновенный поиск процессов по имени или PID.
- **Сортировка**: сортировка таблицы процессов по любому столбцу простым кликом по заголовку.
- **Автообновление**: все данные динамически обновляются с заданным интервалом.
- **REST API**: все метрики также доступны для интеграции через JSON API.

## Архитектура и особенности реализации

Проект построен по принципу «backend на C++ / frontend на TypeScript». Бэкенд в фоновом режиме читает системные файлы из `/proc`, а фронтенд отображает данные, получая их по HTTP.

### Классы и компоненты

#### 1. Бэкенд (C++)

*   **`Processes`**: Мониторинг процессов.
    *   Фоновый поток сканирует `/proc/[pid]/status` для сбора информации (имя, состояние, память, потоки, переключения контекста).
*   **`Cpu`**: Мониторинг загрузки CPU.
    *   Фоновый поток делает два замера из `/proc/stat` с интервалом в 1 секунду.
*   **`Memory`**: Мониторинг памяти.
    *   Фоновый поток читает ключевые параметры из `/proc/meminfo` (MemTotal, MemAvailable, MemFree, Cached).
*   **`MonitorServer`**: HTTP-сервер.
    *   Использует легковесную библиотеку `httplib.h`.
    *   Предоставляет REST API endpoints: `/api/memory`, `/api/cpu`, `/api/processes`.
    *   Раздаёт статические файлы веб-интерфейса (HTML, CSS, JS).
    *   Принимает аргументы командной строки для настройки порта и директории с веб-файлами.

#### 2. Фронтенд (TypeScript/JS)

*   **Периодический опрос**: Каждые 2 секунды через `setInterval` отправляются параллельные `fetch`-запросы к API бэкенда.
*   **Динамическое обновление DOM**: Полученные JSON-данные преобразуются в читаемый формат (например, байты в MB/GB) и обновляют интерфейс.
*   **Интерактивность**:
    *   **Поиск**: Фильтрация строк таблицы по введенному тексту без перезагрузки страницы.
    *   **Сортировка**: Сортировка массива процессов на стороне клиента при клике на заголовок таблицы.

### Используемые технологии

*   **C++17**: Высокая производительность и прямой доступ к системным вызовам.
*   **CMake**: Кроссплатформенная система сборки.
*   **`httplib.h`**: Заголовочная библиотека для встраивания HTTP-сервера.
*   **POSIX / procfs**: Чтение системной информации из виртуальной файловой системы `/proc`.
*   **Многопоточность**: Фоновые потоки для сбора данных (`std::thread`) и `std::shared_mutex` для потокобезопасного доступа к кэшу.
*   **HTML5 / CSS3**: Адаптивный интерфейс с CSS Grid и Flexbox.
*   **TypeScript**: Типизированный JavaScript для надежного кода на фронтенде.

## Установка и сборка

### Требования

- **CMake** (>= 3.10)
- **C++ компилятор** с поддержкой C++17 (g++, clang++)
- **Node.js** (только для перекомпиляции TypeScript, если вы вносите изменения в `script.ts`)

### Инструкция по сборке

1.  **Клонирование репозитория**
    ```bash
    git clone https://github.com/AlexeyL54/mtop.git
    cd mtop
    ```

2.  **Сборка C++ сервера**
    ```bash
    mkdir -p build
    cp -r web build/         # Копируем веб-файлы в директорию сборки
    cd build
    cmake ..
    make
    ```

3.  **Компиляция TypeScript (опционально)**
    Если вы изменили `web/script.ts`, его необходимо скомпилировать в JavaScript:
    ```bash
    # Установите зависимости (один раз)
    npm install

    # Скомпилируйте TypeScript
    npx tsc

    # Скопируйте обновленные файлы в build
    rm -rf build/web
    cp -r web build/
    ```

## Запуск

### Базовый запуск

После сборки выполните из директории `build`:

```bash
./mtop
```

Сервер запустится по адресу `http://localhost:8080`. Откройте этот адрес в браузере.

### Параметры командной строки

Вы можете настроить сервер с помощью аргументов:

```bash
./mtop -p <port> -d <web-directory>
```

- `-p <port>`: Задать порт для HTTP-сервера (по умолчанию: `8080`).
- `-d <dir>`: Указать путь к директории с веб-файлами (по умолчанию: `./web`).
- `-h`: Показать справку.

#### Примеры:

```bash
# Запуск на порту 3000
./mtop -p 3000

# Запуск с пользовательской директорией веб-файлов
./mtop -d /var/www/mtop

# Комбинация параметров
./mtop -p 9090 -d ./my_web_files
```

## Использование веб-интерфейса

1.  **Мониторинг ресурсов**: Верхние карточки показывают текущее состояние памяти и CPU.
2.  **Детали CPU**: Для каждого ядра отображается процент загрузки и визуальная шкала.
3.  **Работа с процессами**:
    *   **Поиск**: Используйте поле "Search by name or PID..." для фильтрации списка.
    *   **Сортировка**: Нажмите на любой заголовок столбца (`PID`, `Name`, `State` и т.д.), чтобы отсортировать таблицу.
    *   **Индикация состояния**: Состояния процессов (`R`, `S`, `D`, `Z`, `T`) подсвечиваются цветными метками для быстрой визуальной оценки.

### Интерпретация состояний процессов

| Состояние | Описание |
|:---------:|:----------|
| **R** | Выполняется (Running) |
| **S** | Ожидание (Sleeping) |
| **D** | Непрерываемое ожидание (Uninterruptible sleep, обычно I/O) |
| **Z** | Зомби (Zombie) |
| **T** | Остановлен (Stopped/Traced) |

## API Endpoints

Сервер предоставляет следующие JSON API, которые вы можете использовать для интеграции с другими инструментами:

*   `GET /api/memory` → Данные о памяти (ключи: `MemTotal`, `MemAvailable`, `MemFree`, `Cached` и др.).
*   `GET /api/cpu` → Данные о CPU (количество ядер и загрузка каждого).
*   `GET /api/processes` → Список всех процессов с полной информацией.

Пример запроса:
```bash
curl http://localhost:8080/api/cpu | jq .
```
